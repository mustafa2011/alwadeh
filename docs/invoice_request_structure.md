# Remaining Suspected Causes

Almost everything has been ruled out.

Current suspicion is limited to:

1.

Canonical XML used during hash generation differs somehow from the original successful signing flow.

OR

2.

ECDSA signature generation differs although using the same source code.

OR

3.

SignedInfo / SignedProperties generated by InvoiceSignatureBuilder differs from the original version.
